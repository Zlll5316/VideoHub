# 📋 一步一步跟着做 - 解决 PGRST205 错误

## ✅ 步骤 1：打开 Supabase Dashboard

1. 访问：https://supabase.com/dashboard
2. 登录你的账号
3. **选择你的项目**（确保选对了项目！）

## ✅ 步骤 2：打开 SQL Editor

1. 在左侧菜单中找到 **"SQL Editor"**
2. 点击进入
3. 你会看到一个代码编辑区域

## ✅ 步骤 3：执行 SQL 脚本

### 方法 A：使用最简单版本（推荐）

1. **打开项目文件**：`最简单-直接创建表.sql`
2. **全选内容**：按 `Cmd/Ctrl + A`
3. **复制**：按 `Cmd/Ctrl + C`
4. **粘贴到 SQL Editor**：按 `Cmd/Ctrl + V`
5. **执行**：
   - 点击右上角的 **"Run"** 按钮
   - 或者按 `Cmd/Ctrl + Enter`

### 执行后应该看到：

✅ 如果成功：
- 显示 "Success" 或绿色的成功提示
- 底部可能显示 "4 rows" 或类似信息

❌ 如果有错误：
- 会显示红色的错误信息
- **请把错误信息告诉我**

## ✅ 步骤 4：验证表已创建

在 SQL Editor 中执行这个查询：

```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('teams', 'team_members', 'team_folders', 'team_videos');
```

**应该返回 4 行**：
- teams
- team_members
- team_folders
- team_videos

如果返回 0 行或少于 4 行，说明表没有创建成功。

## ✅ 步骤 5：刷新 Schema Cache

这是**最关键**的一步！

### 方法 1：等待自动刷新（最简单）
- **等待 3-5 分钟**
- Schema cache 会自动刷新

### 方法 2：手动刷新
1. 在 Supabase Dashboard 中
2. 点击左侧菜单的 **"Settings"**（设置）
3. 点击 **"API"** 标签
4. 向下滚动，找到 **"Reload schema"** 或 **"Refresh schema"** 按钮
5. 点击刷新

### 方法 3：重启项目（如果找不到按钮）
1. Settings → General
2. 找到 **"Restart project"** 选项
3. 重启项目（这会导致 schema cache 刷新）

## ✅ 步骤 6：刷新应用页面

1. 回到应用（localhost:5173）
2. 按 `Cmd/Ctrl + R` 刷新页面
3. 应该就能正常加载了！

## 🔍 如果还是不行

### 检查清单：

- [ ] SQL 脚本执行成功了吗？（没有红色错误）
- [ ] 验证查询返回 4 行了吗？
- [ ] 等待了至少 3 分钟吗？
- [ ] 在 Table Editor 中能看到表吗？

### 如果表创建成功但还是报错：

可能是 Schema Cache 还没刷新，试试：
1. **等待更长时间**（5-10 分钟）
2. **重启 Supabase 项目**
3. **检查浏览器控制台**（F12），看看具体错误
