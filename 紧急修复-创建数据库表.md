# 🚨 紧急修复：创建数据库表

## 错误信息
```
Could not find the table 'public.teams' in the schema cache
```

## ✅ 快速解决方案（3 步）

### 步骤 1：打开 Supabase SQL Editor
1. 访问：https://supabase.com/dashboard
2. 登录并选择你的项目
3. 点击左侧菜单的 **"SQL Editor"**

### 步骤 2：执行 SQL 脚本
1. 打开项目中的文件：**`一键创建团队表.sql`**
2. **全选并复制**所有内容（Cmd/Ctrl + A，然后 Cmd/Ctrl + C）
3. 粘贴到 Supabase SQL Editor 中
4. 点击右上角的 **"Run"** 按钮（或按 `Cmd/Ctrl + Enter`）

### 步骤 3：验证并刷新
1. 执行完成后，你应该看到绿色的成功提示
2. 可以点击左侧菜单的 **"Table Editor"** 验证表已创建
3. 应该能看到 4 个新表：
   - `teams`
   - `team_members`
   - `team_folders`
   - `team_videos`
4. **回到应用，刷新页面**

## ⚠️ 如果执行报错

### 错误 1：表已存在
```
relation "teams" already exists
```
**解决**：忽略这个错误，表已经存在了，直接刷新应用页面即可。

### 错误 2：权限不足
```
permission denied for schema public
```
**解决**：确保你登录的是项目所有者账号，或者联系项目管理员。

### 错误 3：videos 表不存在
```
relation "videos" does not exist
```
**解决**：先创建 `videos` 表（如果还没有的话）。可以暂时注释掉 `team_videos` 表的外键约束。

## ✅ 执行成功后

你应该看到：
- ✅ 4 个表创建成功
- ✅ 索引创建成功
- ✅ RLS 策略创建成功

然后**刷新应用页面**，团队空间应该就能正常工作了！
